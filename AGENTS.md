# 🌍 ГЛОБАЛЬНЫЕ ПРАВИЛА МОНОРЕПОЗИТОРИЯ

## 🧭 Иерархия правил (важно)
1) Корневой `AGENTS.md` — **единый источник истины** для всего репозитория.
2) `packages/**/*/AGENTS.md` — правила для пакетов (shared-код).

Если локальные правила конфликтуют с корневыми — **применяются корневые**, а конфликт нужно исправлять в документации.

## 🚨 ОБЯЗАТЕЛЬНЫЙ РАБОЧИЙ ПРОЦЕСС
Перед написанием кода строго следуй алгоритму:
1. 🔍 **Анализ** → суть, риски, зависимости, edge-cases
2. 📋 **План** → шаги, файлы, слои, команды проверки
3. 💡 **Варианты** → 2-3 подхода с плюсами/минусами (если применимо)
4. ⏸️ **Ожидание** → ЗАПРОСИ разрешение. НЕ пиши код до `ОК` или выбора варианта.
5. 🛠️ **Реализация** → строго по плану. Комментируй только нетривиальные решения.
6. ✅ **Верификация** → команды тестов/линтов, что проверить в UI/API

❌ Запрещено: код без согласования, `any`, любая правка в node_modules.

## 📦 МОНРЕПО-ГРАНИЦЫ
- При добавлении нового пакета обнови `tsconfig.paths` и глобальные скрипты
- Не устанавливай `pnpm` пакеты локально по возможности лучше выкинь предупреждение и запроси установку через `pnpm i` с флагами

## 🛠 ГЛОБАЛЬНЫЕ ИНСТРУМЕНТЫ
- **Менеджер пакетов**: `pnpm`
- **Prettier**: Корневой `.prettierrc`. Форматируй изменённые файлы перед коммитом. Не меняй конфиг без согласования.
- **TypeScript**: `strict: true` везде. `noImplicitAny`, `strictNullChecks`, `exactOptionalPropertyTypes` обязательны.

📌 Язык ответов: русский. Код: английский + комментарии только для сложной логики.

## 🧯 Ошибки и исключения (единый формат)

- **Единый стандарт**: в рантайм-коде приложения не бросаем “сырые” `Error`/`TypeError` и т.п. — используем только **стандартизированные классы ошибок** (пример - `/packages/@biorate/config/src/errors.ts`).
